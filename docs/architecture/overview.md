# OYTAPHP 系统架构

## 整体架构

OYTAPHP 采用分层架构设计，从底层到上层依次为：

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户 PHP 代码                              │
│                    (app/controller/model/...)                    │
├─────────────────────────────────────────────────────────────────┤
│                        Facade 门面层                              │
│   ┌───────┬───────┬───────┬───────┬───────┬───────┬─────────┐  │
│   │  Db   │Cache  │Session│ Config│  Log  │ Event │ Validate │  │
│   └───────┴───────┴───────┴───────┴───────┴───────┴─────────┘  │
├─────────────────────────────────────────────────────────────────┤
│                        PHP 解释器层                               │
│   ┌───────────────┐  ┌───────────────┐  ┌───────────────┐      │
│   │  AST 解析器    │  │  符号表管理    │  │  执行引擎     │      │
│   │ (php-rs-parser)│  │  (dashmap)    │  │  (解释执行)   │      │
│   └───────────────┘  └───────────────┘  └───────────────┘      │
├─────────────────────────────────────────────────────────────────┤
│                        核心服务层                                 │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐          │
│   │ Timer   │ Monitor │ Search  │ Serialize│ Sandbox │          │
│   │ 时间轮   │ 监控    │ 搜索    │ 序列化   │ 沙箱    │          │
│   ├─────────┼─────────┼─────────┼─────────┼─────────┤          │
│   │ Coroutine│  SIMD   │ Cache   │Microserv│  HTTP/3 │          │
│   │ 协程    │ 向量化  │ 缓存    │ 微服务  │ QUIC   │          │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘          │
├─────────────────────────────────────────────────────────────────┤
│                        基础设施层                                 │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐          │
│   │  Axum   │  Tokio  │ SeaORM  │  Redis  │  Moka   │          │
│   │ HTTP服务│ 异步运行│ 数据库  │ 缓存    │ 内存缓存│          │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘          │
└─────────────────────────────────────────────────────────────────┘
```

## 双 Crate 架构

OYTAPHP 采用 Cargo Workspace 组织代码，包含两个 crate：

### oyta_core - 核心运行时

核心运行时二进制，提供完整的 PHP 执行环境：

```rust
// 编译产物
cargo build --release --package oyta_core --bin oyta
// 产物位置：target/release/oyta
```

**主要职责**：
- PHP 代码解析与执行
- HTTP 服务运行
- 数据库连接管理
- 缓存服务
- 热重载支持
- 内嵌服务（WebSocket/定时器/队列）

### oyta_cli - 全局 CLI 工具

全局命令行工具，用于项目创建和管理：

```rust
// 编译产物
cargo build --release --package oyta_cli --bin oyta-cli
// 产物位置：target/release/oyta-cli
```

**主要职责**：
- 创建新项目
- 下载核心二进制
- 管理项目模板

## 核心模块划分

### 解释器模块 (interpreter/)

PHP 代码解析与执行的核心模块：

| 子模块 | 功能 |
|--------|------|
| `engine.rs` | 解释器引擎主入口 |
| `generator.rs` | Generator/yield 支持 |
| `fiber.rs` | Fiber 协程支持 |
| `enum_type.rs` | 枚举类型支持 |
| `readonly.rs` | 只读属性支持 |
| `php8_features/` | PHP 8.x 新特性 |
| `value.rs` | PHP 值类型定义 |

### 定时器模块 (timer/)

多级时间轮定时器实现：

| 子模块 | 功能 |
|--------|------|
| `wheel.rs` | 时间轮核心实现 |
| `level.rs` | 时间轮层级（4级） |
| `slot.rs` | 时间槽位管理 |
| `task.rs` | 定时任务定义 |
| `scheduler.rs` | 任务调度器 |

### 监控模块 (monitor/)

实时性能监控面板：

| 子模块 | 功能 |
|--------|------|
| `collector.rs` | 指标收集器 |
| `dashboard.rs` | 监控面板 |
| `metrics.rs` | 指标定义 |

### 缓存模块 (cache/)

多级缓存系统：

| 子模块 | 功能 |
|--------|------|
| `manager.rs` | 缓存管理器 |
| `driver.rs` | 缓存驱动接口 |
| `redis_driver.rs` | Redis 驱动 |
| `multi_level.rs` | 多级缓存 |
| `warmup/` | 缓存预热 |

### 协程模块 (coroutine/)

原生协程调度器：

| 子模块 | 功能 |
|--------|------|
| `scheduler.rs` | 协程调度器 |
| `worker.rs` | 工作线程 |
| `task.rs` | 协程任务 |
| `channel.rs` | 协程通道 |
| `sync.rs` | 同步原语 |

### 搜索模块 (search/)

全文搜索引擎：

| 子模块 | 功能 |
|--------|------|
| `index.rs` | 倒排索引 |
| `query.rs` | 查询解析 |

| `tokenizer.rs` | 分词器 |

### SIMD 模块 (simd/)

SIMD 向量化加速引擎：

| 子模块 | 功能 |
|--------|------|
| `engine.rs` | SIMD 引擎核心 |
| `string_ops.rs` | 字符串操作 |
| `array_ops.rs` | 数组操作 |
| `json_parse.rs` | JSON 解析 |
| `hash.rs` | 哈希计算 |

### 序列化模块 (serialize/)

零拷贝高性能序列化：

| 子模块 | 功能 |

### 沙箱模块 (sandbox/)

## 数据流

### 请求处理流程

```
HTTP 请求
    │
    ▼
┌─────────────┐
│ Axum Server │ ← 接收 HTTP 请求
└─────────────┘
    │
    ▼
┌─────────────┐
│ Router      │ ← 路由匹配
└─────────────┘
    │
    ▼
┌─────────────┐
│ Middleware  │ ← 中间件链（洋葱模型）
└─────────────┘
    │
    ▼
┌─────────────┐
│ Interpreter │ ← 执行 PHP 控制器代码
└─────────────┘
    │
    ▼
┌─────────────┐
│ Facade      │ ← 调用 Db/Cache/Session 等
└─────────────┘
    │
    ▼
┌─────────────┐
│ Response    │ ← 构建响应
└─────────────┘
    │
    ▼
HTTP 响应
```

### 热重载流程

```
文件变更
    │
    ▼
┌─────────────┐
│ Notify      │ ← 文件监听器检测变更
└─────────────┘
    │
    ▼
┌─────────────┐
│ Parser      │ ← 重新解析 PHP 文件
└─────────────┘
    │
    ▼
┌─────────────┐
│ SymbolTable │ ← 更新符号表
└─────────────┘
    │
    ▼
┌─────────────┐
│ Router      │ ← 如需要，重建路由表
└─────────────┘
    │
    ▼
热重载完成
```

## 性能优化策略

### 编译时优化

| 策略 | 说明 |
|------|------|
| AST 缓存 | 解析后的 AST 缓存在 dashmap 中 |
| 路由编译 | 路由表编译为 Trie 前缀树 |
| 配置缓存 | 配置序列化为二进制缓存 |

### 运行时优化

| 策略 | 说明 |
|------|------|
| 多级缓存 | moka L1（内存）+ Redis L2 |
| 连接池 | 数据库/Redis 连接池 |
| 零拷贝 | bytes + axum 直接写入 socket |
| SIMD 加速 | 字符串/数组/JSON 操作向量化 |

## 安全机制

| 安全项 | 实现方式 |
|--------|----------|
| SQL 注入防护 | SeaORM 参数化查询 |
| XSS 防护 | 模板引擎默认 HTML 转义 |
| CSRF 防护 | Session + Token 验证 |
| 路径遍历防护 | 静态文件服务禁止 `../` |
| 加密 | AES-GCM 加密敏感数据 |
